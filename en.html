What is AIDY?

AIDY (AI Designed for You) is an open-source AI agent team platform for solo entrepreneurs. Domain-expert AI agents work autonomously under human supervision, delivering human-level quality output.

Instead of treating AI as a tool, AIDY treats AI as a team — each agent has its own role (SOUL), shared goals (POLARIS), and domain knowledge.

Key Features

  • 4 Specialized Agents — Researcher, Content Writer, Executor, Analyst
  • AIDY Core Orchestrator — Coordinates agents via WebSocket gateway (9 modules)
  • Human-in-the-loop — Approval workflow via Discord
  • 11-Page Dashboard — Real-time monitoring, task management, agent status
  • Asana Integration — Task management as the system's "brain"
  • Discord Integration — Gateway Bot, slash commands, per-agent webhook profiles
  • Supabase Backend — PostgreSQL + Storage + TipTap/Novel Documents
  • n8n Workflows — 12 automation workflows as the system's "nervous system"
  • Multi-Runtime Adapter — OpenClaw + Claude Code (DEC-051, extensible)

Design Philosophy: AX (AI Experience)

Agents

AIDY runs 4 specialized agents + AIDY Core orchestrator:

Agent Role Personality
AIDY Core Orchestrator — task routing, agent control (Next.js server) Strategic, systematic
Researcher Data collection, market research, keyword analysis Detail-oriented, evidence-based
Content Writer Marketing asset production (blog, social media) Creative, accuracy-focused
Executor Browser automation, publishing, posting Cautious, error-sensitive
Analyst Performance monitoring, reporting, ROI analysis Objective, data-driven

Each agent runs as an isolated OpenClaw process (--profile separation) on local PC.

Quick Start

Prerequisites

  • Node.js >= 18
  • Asana account (Starter plan, 1 seat)
  • Discord server with bot application
  • Supabase project (free tier OK)
  • OpenClaw installed locally

Installation

# 1. Clone the repository
git clone https://github.com/sinwoo/aidy-dashboard.git
cd aidy-dashboard

# 2. Run setup (installs dependencies, creates .env.local)
node scripts/setup.js

# 3. Configure environment variables
# Edit dashboard/.env.local with your API keys
# See .env.example for all available options

# 4. Start development server
npm run dev
# Dashboard available at http://localhost:8093

# 5. Verify installation
npm run verify

Production

# Build and start with PM2
npm run build
npm run start

# View logs
npm run logs

# Stop
npm run stop
